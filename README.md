# 🪴 Field Notes & Digital Garden

> **🚧 Status: v0.5 (Active)** > This repository is my personal scratchpad, digital garden, and "learning in public" workspace. It is where I document messy experiments, summarize academic papers, and store reusable code snippets before they graduate to production repositories.

## 🧠 Philosophy
This repo operates on the principle of **Trunk-Based Knowledge Management**. 
It is optimized for VS Code, relative markdown links, and global search. There are no complex tagging trees or nested folder labyrinths. If it is a thought, a theory, or a test, it lives here.

## 🗺️ Navigation (Directory Structure)

The repository is kept intentionally flat and is divided into five core domains:

* **`assets/`**
    * The central dump for all screenshots, diagrams, and static files referenced in the markdown notes. 
* **`01-journal/`**
    * Chronological, time-based logs. Daily thoughts and raw brainstorming.
* **`02-snippets/`**
    * The technical arsenal. Reusable, single-file code chunks, Bash commands, Python tricks, and configuration templates.
* **`03-concepts/`**
    * The 'public' knowledge vault. Deep dives into theoretical frameworks, linguistics literature reviews, and summaries of foundational academic papers.
* **`04-drafts/`**
    * The output pipeline. Markdown staging grounds for upcoming blog posts and articles before they are published.
* **`05-experiments/`**
    * The "Labs" sandbox. Self-contained, multi-file mini-projects (e.g., semantic segmentation tests) that require their own scripts, Jupyter notebooks, and local readmes.
